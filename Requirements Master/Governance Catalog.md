# 2. License Governance Filters

To prevent legal risk and keep UAWOS IP safe, the platform enforces the following licensing checks in the build and integration lifecycles:

| License Class | Status | Examples | Requirements |
| :--- | :--- | :--- | :--- |
| **Permitted** | Approved | MIT, Apache 2.0, BSD-3, ISC | Standard adoption allowed without restriction. |
| **Conditional** | Review Required | LGPL, EPL, ELv2 | Allowed only if wrapped or isolated; direct static linking requires Strategy Council approval. |
| **Prohibited** | Blocked | AGPLv3, SSPL, Proprietary SaaS (unapproved) | Strictly blocked. Building dependencies on these licenses triggers CI build failures. |

# 3. Data Lineage & Metadata Controls

Data lineage is captured to ensure all data transformations can be traced to their originating inputs.

* **Execution Ingestion**: The system utilizes **OpenLineage** to capture data lineage events from `dbt` transforms and `Meltano` pipelines.
* **Metadata Storage**: Core logs and relationships are stored in **Marquez**, exposing interactive lineage charts to trace data lineage.
* **Catalog Governance**: **OpenMetadata** serves as the metadata directory to catalog tables, models, vector stores, and metrics.

# 4. Policy Engine Rules (OPA / Rego Baseline)

The Governor Agent evaluates all high-risk actions against the OPA policy repository. The baseline policy checks include:

```rego
package uawos.authz

default allow = false

# Allow reads without intervention
allow {
    input.action.type == "READ"
}

# Require human approval for financial actions or write steps on production DBs
allow {
    input.action.type == "WRITE"
    input.action.risk_level < 3
    input.subject.trust_score > 0.85
}

# Block execution of tools containing unpermitted licenses
allow = false {
    input.dependency.license == "AGPLv3"
}
```

# 5. Continuous Governance Auditing

* **Lineage Audits**: System logs verify the presence of an unbroken chain from objective intent to final value reports.
* **Audit Trails**: All decision outputs generated by the Strategy and Architecture Councils are logged as immutable assets inside the Knowledge Graph.
