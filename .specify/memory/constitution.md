# Universal AI Workforce Operating System (UAWOS) Constitution

This Constitution is the highest governing authority of the UAWOS platform. All standards, architectures, implementations, packs, engines, graphs, agents, and integrations SHALL conform to this Constitution. Where conflicts exist, this Constitution SHALL prevail.

## Constitutional Mission & Vision

- **Constitutional Mission**: Transform any objective into measurable value through governed, explainable, continuously learning execution.
- **Constitutional Vision**: Create a universal operating system for human and AI workforce coordination that can operate across any domain, industry, organization, or execution environment.

## Core Principles

### Principle 1: Objective Centricity
All execution SHALL originate from an Objective. Objective is the universal abstraction.

### Principle 2: Value Realization
Value Realization is the ultimate measure of success. Outputs and outcomes are insufficient on their own; value realization is required.

### Principle 3: Governance Supremacy
Governance supersedes execution. No execution capability may bypass governance. Governance is the supreme control plane.

### Principle 4: Knowledge Asset Preservation
Knowledge is an organizational asset. Knowledge SHALL be preserved, governed, and continuously improved.

### Principle 5: Accountability
Humans remain accountable. Responsibility may be delegated; accountability may not.

### Principle 6: Computed Trust
Trust is earned. Trust SHALL be computed. Trust SHALL NOT be manually assigned.

### Principle 7: Governed Autonomy
Autonomy is earned. Autonomy SHALL increase only through demonstrated trust and governance approval.

### Principle 8: Context Precedence
Context precedes decision. All decisions SHALL be context-aware.

### Principle 9: Evidence-Based Recommendations
Evidence precedes recommendation. Recommendations SHALL be evidence-based.

### Principle 10: Absolute Traceability
Traceability is mandatory. Every outcome SHALL trace to its originating objective.

## System Architecture & Integration Controls

### Constitutional Graph Model
The platform operates through a federated set of graphs. No single graph is authoritative for all enterprise knowledge:
- **Objective Graph**: Stores objectives, outcomes, plans, and workflows.
- **Knowledge Graph**: Stores knowledge, evidence, and claims.
- **Agent Graph**: Stores agents, teams, and capabilities.
- **Resource Graph**: Stores resources, capacity, and allocations.
- **Policy Graph**: Stores policies, risks, and approvals.
- **Value Graph**: Stores metrics, outcomes, and value realization.
- **Portfolio Graph**: Stores portfolios and strategic themes.
- **Context Graph**: Stores context models and relationships.

### Constitutional Workforce & Agent Classes
The workforce consists of Humans, Agents, Teams, and Agent Teams. Every compliant implementation MUST support the following agent classes:
- **Planner**: Designs technical plans and feature execution routes.
- **Orchestrator**: Schedules and coordinates workflows.
- **Executor**: Executes individual actions.
- **Reviewer**: Evaluates execution outputs and code quality.
- **Governor**: Enforces policy compliance and risk assessments.
- **Learner**: Translates execution signals into model or operational improvements.
- **Knowledge Manager**: Governs memory, ontology, and knowledge assets.

### Memory & Knowledge Rules
- **Memory**: Must be append-only, support overlays, preserve history, and respect retention policies. Deletion is never authoritative.
- **Knowledge**: Must be governed, attributable, versioned, traceable, and possess confidence values. Contradictory knowledge is preserved before reconciliation.

## Development Workflow & SDD Commands

This project uses Spec-Driven Development (SDD) powered by GitHub Spec Kit. Developers and AI coding agents (Claude Code) interact using the following slash commands:

- `/speckit-constitution` - Establish and update project principles in this file.
- `/speckit-specify` - Create baseline feature specifications.
- `/speckit-plan` - Design technical plans and architectures.
- `/speckit-tasks` - Generate actionable tasks from plans.
- `/speckit-implement` - Execute implementation and code changes.
- `/speckit-clarify` (optional) - Ask clarifying questions to de-risk ambiguous requirements.
- `/speckit-checklist` (optional) - Generate quality checklists to validate requirements.
- `/speckit-analyze` (optional) - Cross-check artifact consistency and alignment.

### Git Conventions
- **Branch Naming**: Follow the sequence `[###-feature-name]` (e.g., `001-setup-constitution`).
- **Commit Messages**: Must clearly state the component and action (e.g., `docs: configure UAWOS constitution v1.0.0`).
